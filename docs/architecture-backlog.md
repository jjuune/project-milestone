# 📌 PWA + Supabase 기술 검토 및 아키텍처 백로그

본 문서는 **PITSTOP (Milestone)** 프로젝트의 MVP 및 V1.5 단계에서 PWA(프로그레시브 웹 앱)와 Supabase 백엔드를 연동할 때 반영해야 할 핵심 구현 로직과 기술 검토 사항을 기록합니다.

---

## 1. Supabase 데이터 연동 및 PWA 오프라인 캐싱
대시보드의 핵심 UX인 "오늘 당장 해야 할 조치"를 음영 지역(지하 주차장 등)에서도 끊김 없이 제공하기 위한 데이터 처리 로직입니다.

- [ ] **App Shell 캐싱**: 서비스 워커(Service Worker)를 통해 앱의 기본 UI 구조(HTML, CSS, JS)를 기기에 로컬 캐싱하여 실행 즉시(Instant) 대시보드 틀이 뜨도록 구현.
- [ ] **오프라인 입력 및 동기화 (Offline-First UX)**: 
  - 네트워크 음영 지역에서 차량 정비/주행거리 이벤트 입력 시, Supabase 실패 처리를 막고 로컬 스토리지(`IndexedDB` 또는 캐시)에 임시 저장.
  - 브라우저의 `online` 이벤트 감지 및 서비스 워커의 `Background Sync`를 활용하여 네트워크 복구 시 Supabase 테이블에 자동 동기화(`Upsert`) 수행.

## 2. '10초 퀵 입력 UX'를 위한 OS별 파일/카메라 대응
이벤트 타임라인 등록 시 영수증 및 내역 사진을 현장에서 바로 첨부하기 위한 네이티브 인터페이스 연동 로직입니다.

- [ ] **네이티브 카메라 핫링크**: 
  - HTML 내 `<input type="file" accept="image/*" capture="environment">` 태그 활용.
  - 안드로이드/iOS PWA 환경 모두에서 별도 서드파티 라이브러리 없이 기기 카메라 앱을 즉시 구동하거나 사진 앨범 접근 인터페이스 제공.
- [ ] **Supabase Storage 버킷 연동**: 
  - 촬영된 이미지를 `Supabase Storage` 내 `receipts` 버킷에 즉시 업로드.
  - 업로드 성공 후 반환된 `Public URL`을 해당 차량의 `Event Timeline` 테이블 레코드 필드에 매핑하여 저장 (추후 V2 OCR 도입을 위한 데이터 표준화).

## 3. 선제적 알림(Push) 시스템 및 OS별 분기 대응
보험 만기일, 정기검사, 소모품 교체 주기 도래 시 사용자에게 백그라운드 알림을 보내기 위한 Web Push 아키텍처입니다.

- [ ] **Web Push Subscription 관리**: 
  - 유저가 PWA 설치 후 알림 권한을 동의하면, 기기 고유의 `Subscription(구독 토큰)` 생성.
  - 이 토큰을 Supabase 데이터베이스의 `user_subscriptions` 테이블에 매핑하여 관리.
- [ ] **Supabase Edge Functions 기반 스케줄러**: 
  - Supabase Edge Functions(또는 Cron 툴)를 활용해 매일 새벽 차량별 상태 데이터(`D-12`, 주행거리 임계치) 연산.
  - 알림 대상 발생 시 저장된 웹 푸시 토큰을 이용해 기기로 백그라운드 푸시 알림 발송.
- [ ] **iOS 제약사항 대응 가이드**: 
  - iOS는 홈 화면에 PWA를 추가한 상태(iOS 16.4 이상)에서만 웹 푸시 수신이 가능함.
  - 따라서 최초 웹 진입 시 **"iOS 사용자를 위한 홈 화면 추가 수동 가이드(공유 버튼 안내 툴팁)"** 노출 로직 선행 필수.

## 4. NFC 딥링크 및 PWA 앱 진입 흐름
차량 내부(콘솔 트레이 등)에 부착된 NTAG215 스캔 시, PWA 앱으로 직접 진입하거나 유도하는 로직 검토입니다.

- [ ] **안드로이드 PWA 직접 실행**:
  - NFC에 PWA 특정 차량 상세 URL(`https://your-pitstop-pwa.com{vehicle_id}`) 탑재.
  - 안드로이드 기기는 앱이 이미 설치된 경우 브라우저를 거치지 않고 설치된 PITSTOP PWA 앱이 standalone(전체 화면)으로 즉시 실행되도록 대응.
- [ ] **iOS Safari 우회 흐름**:
  - iOS 특성상 NFC 스캔 시 무조건 Safari 브라우저로 1차 진입함.
  - 이미 홈 화면에 앱을 추가해 둔 상태라면 Safari 상단 "앱에서 열기" 스마트 배너를 활용하거나, 미설치 시 설치 가이드로 자연스럽게 랜딩시키는 예외 처리 로직 적용.

## 5. [추가 반영] Supabase Realtime을 활용한 멀티 유저 동기화
1~5대의 차량을 여러 명의 담당자 혹은 현장 직원이 공동 관리할 때 발생할 수 있는 데이터 정합성 문제를 해결하기 위한 실시간 UX 로직입니다.

- [ ] **실시간 상태 동기화 (Supabase Realtime)**: 
  - 대시보드 및 차량 카드 화면에 Supabase의 `psql_synced_objects_notifies` 레벨 리스너 적용.
  - 동료 직원이 현장에서 다른 차량의 주행거리를 입력하거나 이벤트를 등록하는 즉시, 내가 보고 있는 대시보드 화면이 새로고침 없이 실시간으로 게이지 바와 상태값(🟢🟡🔴)이 갱신되도록 실시간 구독(Subscribe) 로직 구현.


## 6. [추가 반영] 멀티 유저 대상 다중 푸시 알림 발송 로직
소규모 사업장의 특성상 1대의 차량을 여러 명의 관리자(담당 직원, 대표자 등)가 함께 모니터링하므로, 이벤트 발생 시 관계자 전원에게 알림을 동의 전송하기 위한 아키텍처 구조입니다.

- [ ] **1대다(1:N) 구독 토큰 매핑 구조 설계**:
  - 특정 차량 ID 또는 회사(조직) ID 하나에 여러 사용자의 웹 푸시 구독 토큰(`Subscription Token`)이 종속될 수 있도록 Supabase 테이블 관계 설정.
  - 한 유저가 모바일과 PC 양쪽에서 PWA 알림을 동의한 경우, 해당 유저 ID 하위에 기기별 토큰이 각각 생성 및 적재되도록 처리.
- [ ] **Edge Functions 내 비동기 루프 발송 처리**:
  - 알림 이벤트(정기검사 임박 등) 트리거 시, 연관된 모든 관계자의 구독 토큰 목록을 Supabase DB에서 일괄 조회.
  - 백엔드 서버(Edge Functions) 내에서 `Promise.all()` 등 비동기 병렬 처리 로직을 구현하여, 조회된 모든 토큰 기기로 동시에 웹 푸시(Web Push) 신호 발송.
- [ ] **디바이스별 중복 알림 필터링 정책 정의**:
  - 한 유저가 다중 기기(스마트폰, 노트북 등)를 등록했을 때 모든 기기에 알림을 동시에 노출할 것인지, 혹은 최종 활성화된 주 기기에만 발송할 것인지에 대한 예외 처리 비즈니스 로직 수립.
